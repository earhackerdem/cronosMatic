## Why

The backend exposes Google OAuth endpoints (Ticket 003, capability `google-oauth`) but the React/Vite frontend still only supports email + password. Without a UI entry point and a callback route to capture the issued JWT, users cannot actually log in with Google. Ticket 004 closes that gap so the social-login flow is end-to-end usable.

## What Changes

- Add a "Continue with Google" button to `/login` and `/signup`. Clicking it sends the browser to `${VITE_API_URL}/api/v1/auth/google/login` (full-page navigation, not `fetch`) so the backend can perform the 302 to Google.
- Add a new TanStack Router route `/auth/callback` that:
  - Reads `access_token` from the URL search params, stores it in `localStorage` under the existing `access_token` key, invalidates the `currentUser` query, and navigates to `/`.
  - Reads an optional `error` search param and, when present, navigates to `/login` and surfaces a toast via `useCustomToast`.
  - Strips the token from the URL before/while rendering (no flicker, no token in history).
- Extend `useAuth` with a small helper (`loginWithGoogleToken(token: string)`) that encapsulates the localStorage write + query invalidation so the callback route reuses existing state management rather than duplicating it.
- Add a `VITE_GOOGLE_LOGIN_ENABLED` (or equivalent) build-time flag so the Google button is hidden when the backend is not configured for Google OAuth — keeps parity with the backend's HTTP 503 behavior.

## Capabilities

### New Capabilities
- `google-oauth-frontend`: UI entry points (login/signup buttons), the `/auth/callback` route that captures the backend-issued JWT, and the auth-state integration that logs the user in and redirects to the dashboard.

## Impact

- **Frontend code**:
  - `frontend/src/routes/login.tsx`, `frontend/src/routes/signup.tsx` — add Google button.
  - `frontend/src/routes/auth.callback.tsx` (new) — capture token, log in, redirect.
  - `frontend/src/hooks/useAuth.ts` — add `loginWithGoogleToken` helper.
  - `frontend/src/components/Common/GoogleLoginButton.tsx` (new) — shared Shadcn-styled button.
  - `frontend/src/routeTree.gen.ts` — auto-regenerated by TanStack Router.
- **Frontend config**:
  - `frontend/.env`, `.env.example` — add `VITE_GOOGLE_LOGIN_ENABLED` (default `false`).
- **Tests**:
  - New Playwright spec `frontend/tests/google-login.spec.ts` covering button visibility, callback success, and callback error handling (Google OAuth itself is not exercised — we navigate directly to `/auth/callback?access_token=…` and to `/auth/callback?error=…`).
- **No backend changes**: the backend contract from Ticket 003 is already in place; this change only consumes it.
